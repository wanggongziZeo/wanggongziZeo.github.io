前言

使用github和hexo搭建个人博客是比较简单的方式之一，也是很多程序员采用的主要方式，也有很多好处：

  • 静态文件，访问速度快；
  • 方便易操作，是在github平台上进行管理与开发，并且使用markdown进行编写；
  • 可以使用免费的github的账户名，也可以自己申请域名；
  • 有很多出题可供选择，且修改容易。

平台及工具

  • 有自己的github账户，对于程序员来说，这个基本人手一个；
  • windows版本的git（默认win10）安装；
  • node、npm安装，这两个是要用到，不需要对其有很多了解；
  • hexo。
    上述准备充分后，就跟着步骤一步步来，不出意外就可以搭建出个人博客了。

创建个人博客主页

首先应该知道，个人博客依托于github进行管理，而Hexo是基于github的博客发布工具，所以，博客搭建需要先在github上进行部署及设置，然后使用hexo进行编写以及管理。

  1. 创建仓库
    首先创建一个名为github用户名.github.io的仓库，比如我的github名字为wanggongziZeo，那么我的仓库名为wanggongziZeo.github.io。注意，这个仓库名必须是这种形式，不能改变，之后博客中所有的文章都是在这个仓库中进行管理的，而且个人博客访问链接也就是这个。
    创建成功后，可以打开setting设置看一下，里面会有一些关于博客界面和主题的介绍，很简单。
  2. 域名
    如果想使用其他域名进行访问，可以自己申请，阿里云不错，可以去看看。我个人是没有绑定域名的，此处不做介绍。
  3. 配置SSH Key
    配置ssh key是为了解决本地文件和服务器端文件相连接，如果没有进行配置，每次提交时都需要从输入账户和密码，相当麻烦。
    git安装之后，打开git bash，输入ssh-keygen -t rsa -C “邮件地址”，然后按照要求操作，一般是选择默认，同时询问是否需要输入密码，此处直接回车，不需要输入密码，最终会生成一个文件.ssh在user文件夹下，使用写字板打开文件.ssh/id_rsa.pub文件并复制所有内容，打开github主页，进入setting->SSH and GPG key，然后点击New SSh key，并将刚才的内容复制进来，名字可以随便取，保存。之后混默认为使用这个SSH进行配置。
    这些做完之后，可以输入ssh -T git@github.com进行测试，如果出现Are you sure you want to continue connecting (yes/no)?，输入yes，然后会看到：Hi wanggongziZeo! You’ve successfully authenticated, but GitHub does not provide shell access.，说明SSH配置成功了。

    最后，还需要配置个人的github账号和密码：
    git config --global user.name “wanggongziZeo”// 你的github用户名，非昵称
    git config --global user.email "xxx@qq.com"// 填写你的github注册邮箱

hexo配置

hexo用来部署博客，主要是将本地的markdown文件提交到个人主页，下面从hexo安装，博客初始化，主题安装，以及博客提交来说明如何使用。

  1. hexo安装
    npm install -g hexo即可
  2. 初始化
    在电脑某个盘中新建名为Blog的的文件夹，如E:/Blog，该文件夹下将存放代码以及配置信息，比较重要。
    cd E:/Blog
    hexo init
    初始化后，会在Blog文件夹下下载一些文件，主要是博客界面的配置信息，以及一些主题（themes文件夹下），本地要编写博客的存放地址（source），同时会发现有一个文件_config.yml，该文件是博客界面一些相关信息的配置文件，首先需要配置其中关于deploy的部分：
    deploy:
      type: git
      repository: git@github.com:wanggongziZeo/wanggongziZeo.github.io.git（此处对应更改）
      branch: master（如果没有使用其他域名，此处branch统一使用master，其他主题配置也是如此），
    其他信息可以对应着生成的博客进行更改，。进入themes文件夹下会发现有一个名为landscape的文件夹，这个文件夹就是hexo默认的主题，里面有该主题的配置文件，同时发现，也有一个_config.yml，与上面的区分开：这个是博客界面的配置信息，上面那个是博客一些主题信息配置，如名字，文字显示等，同时也提供了一个readme.md文件，该文件就类似于自己写的博客。
    到此时，所有的工作已经做完了，github配置，hexo配置，接下里就是怎样将本地的博客如hexo提供的readme.md文件提交到github即个人主页，很简单，只需要两步：
    hexo g //generate 生成博客
    hexo d //deploy 部署博客
    之后打开个人主页如wanggongziZeo.github.io，就会看到已经生成的博客。

    每次重新编辑博客后，不一定需要每次部署到github上，此时可以先hexo g生成，然后hexo s，通过本地预览，打开浏览器访问 http://localhost:4000就可看到内容。

写博客

定位到我们的hexo根目录，执行命令：
hexo new 'my-first-blog’
hexo会帮我们在_posts下生成相关md文件，只需要打开这个文件就可以开始写博客了，后续操作和上述类似。
至于用什么工具编写markdown文件，有很多选择，个人推荐vs-code，方便。

主题更换

网上有很多相关的hexo主题，可以自己选择喜欢的进行更改部署。以主题hexo-theme-yilia为例，具体操作如下：
cd E:/Blog/themes
git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia
下载后的主题都在themes文件夹下，修改根目录下_config.yml中的theme: landscape改为theme: yilia，然后重新执行hexo g来重新生成。
如果出现一些莫名其妙的问题，可以先执行hexo clean来清理一下public的内容，然后再来重新生成和发布。
